## [OpenAI 发布 GPT-5.6 模型，提供三种尺寸并带来新能力](https://openai.com/index/gpt-5-6/) ⭐️ 9.0/10

OpenAI 发布了其最新旗舰 AI 模型 GPT-5.6，该模型提供三种尺寸：Luna、Terra 和 Sol，每种尺寸对应不同的性能等级和定价。 此次发布声称在复杂任务上实现了卓越的性能和效率，在快速发展的大型语言模型市场中构成了一项具有竞争力的更新，并直接挑战了 Anthropic 的 Claude 系列等现有模型。 这些模型按每百万输入/输出 token 定价，最小的 Luna 模型定价为 1/6 美元，最大的 Sol 模型定价为 5/30 美元；值得注意的是，开发者指南强调了其增强的意图理解能力以及保留图像原始尺寸的能力。

**背景**: GPT-5.6 是 OpenAI 之前 GPT-5 系列模型的后续版本，OpenAI 是生成式人工智能领域的领先公司。此类大型语言模型 (LLM) 基于海量数据集进行训练，能够生成类似人类的文本并执行编码、分析和图像理解等任务。像 ARC-AGI-3 这样的基准测试分数通常用于比较这些模型的高级推理和问题解决能力。

## [尽管多数议员反对，欧洲议会仍通过'聊天监控 1.0'法案](https://www.patrick-breyer.de/en/eu-parliament-greenlights-chat-control-1-0-breyer-our-children-lose-out/) ⭐️ 8.0/10

欧洲议会批准了'聊天监控 1.0'法规，允许美国科技公司在无需搜查令的情况下对私人信息进行大规模扫描，有效期至 2028 年。尽管大多数投票的欧洲议会议员（314 票反对，276 票赞成）反对此决定，但由于否决动议未能获得所需的 361 票绝对多数而得以通过。 该立法代表了欧盟数字隐私的重大倒退，允许对 Instagram、Discord 和 Gmail 等主要平台的私人通讯进行监控。它开创了通过程序规则绕过民主多数意愿的先例，并引发了以保护儿童为名将大规模监控常态化的担忧。 该法规在夏季休会前最后一次会议日通过'默认接受'程序规则被重新激活，要求所有欧洲议会议员（不仅仅是投票者）的绝对多数才能否决；有 113 名议员缺席。它允许使用'客户端扫描'技术，该技术在发送前在用户设备上分析消息内容。

## [Bun JavaScript 运行时从 Zig 重写为 Rust](https://simonwillison.net/2026/Jul/8/rewriting-bun-in-rust/#atom-everything) ⭐️ 8.0/10

Bun 的创始人 Jarred Sumner 宣布将 Bun JavaScript 运行时从 Zig 完全重写为 Rust，这一过程在人工智能编程代理的大量帮助下于 11 天内完成，并已通过 Claude Code 部署到生产环境。 此次重写表明，现代人工智能代理可以使以前不可想象的大规模重构变得可行，可能会改变业界对遗留代码库维护和语言迁移的看法。它还凸显了像 Rust 这样的内存安全语言与像 Zig 这样的手动内存管理系统在复杂项目中的实际权衡。 此次重写的动机是 Bun 的 Zig 代码库中持续存在的内存安全错误，如释放后使用，而 Rust 的编译器和 RAII 系统可以防止此类问题。该项目利用 Bun 现有的 TypeScript 测试套件作为一致性测试套件，自动化了大部分移植工作，整个工作估计消耗了价值 16.5 万美元的 API 令牌。

rss · Simon Willison · 7月8日 23:57

**背景**: Bun 是一个基于 JavaScriptCore 构建的流行、高性能 JavaScript 运行时和工具包。Zig 是一种系统编程语言，以其性能和手动内存管理而闻名，而 Rust 则是一种专注于内存安全和并发且无垃圾回收的语言。“代理式工程”指的是使用协调的人工智能代理（通常由大型语言模型驱动）来自主执行复杂的软件开发任务。

## [Colibrì 项目在 32GB 内存的普通计算机上运行 GLM 5.2 大语言模型](https://github.com/JustVugg/colibri) ⭐️ 7.0/10

一位开发者通过创建一个使用 int4 量化并从磁盘流式传输模型专家的推理引擎，成功在仅有 32GB 内存的个人笔记本电脑上运行了 Z.AI 的 7440 亿参数 GLM 5.2 混合专家模型，实现了每秒 0.1 个 token 的功能性但缓慢的推理速度。 该项目展示了在消费级硬件上（即使没有 GPU）运行超大规模前沿模型的可行性，尽管性能存在显著折衷，但有望为实验和开发提供更多人接触强大 LLM 的机会。 Colibrì 引擎是一个单独的 C 文件，它将模型的密集部分（约 170 亿参数）以 int4 量化形式常驻内存（约 9.9 GB），并通过 LRU 缓存从磁盘流式传输 21,504 个路由专家，但警告这可能会导致显著的 SSD 磨损。

hackernews · vforno · 7月9日 08:05 · [社区讨论](https://news.ycombinator.com/item?id=48842459)

**背景**: GLM 5.2 是 Z.AI 的旗舰长时任务模型，拥有庞大的 7440 亿参数混合专家架构，这意味着每个 token 只有部分参数被激活。诸如多 token 预测（MTP）和动态稀疏注意力（DSA）等技术被用于优化长上下文推理。量化（例如将模型权重从 16 位转换为 4 位整数 int4）是减小大型模型内存占用的常用方法。
